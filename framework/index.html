{% if phase.name == "Discover" %}

Key Documents

  • Problem Canvas — Problem, affected users, current workarounds, cost of inaction (1 page)
  • Stakeholder Map — Who influences, decides, and is affected
  • Constraint Register — Budget, time, regulatory, technical constraints

Exit criteria: Stakeholders agree on the problem and it's worth solving.

{% endif %} {% if phase.name == "Design" %}

Key Documents

  • Solution Brief — Vision, scope, high-level approach, success metrics (2–3 pages)
  • Architecture Diagram — Components, integrations, data flow
  • Decision Log (ADR) — Key decisions and rationale
  • Data Map — What data, where it lives, who owns it
  • Risk & Assumptions Log — RAID — what could go wrong

Exit criteria: Team and client agree on what's being built and why.

{% endif %} {% if phase.name == "Develop" %}

Key Documents

  • Product Backlog — Prioritised list of features and user stories
  • Sprint Plan — What gets built this cycle
  • Definition of Done — Shared agreement on when something is complete
  • Test Scenarios — Key scenarios to validate against

Exit criteria: Working software that passes agreed test scenarios.

{% endif %} {% if phase.name == "Implement" %}

Key Documents

  • Go-Live Checklist — Everything that must be true before launch
  • Cutover Plan — Steps to move from old to new, with rollback option
  • User Guide / Runbook — How to use and operate the solution
  • Training Plan — Who needs to know what before go-live

Exit criteria: Solution is live, users can operate it, rollback plan exists.

{% endif %} {% if phase.name == "Operate" %}

Key Documents

  • Operating Model — Roles, responsibilities, escalation paths
  • Support Playbook — How to handle common issues and incidents
  • Monitoring Dashboard — Key health metrics visible at a glance
  • Retrospective Log — What worked, what didn't, what to improve

Exit criteria: Team can support and evolve the product without the original architect.

{% endif %}

The Lean Rules

1

One source of truth — pick one tool and keep everything there

2

Living documents only — if it won't be updated, don't create it

3

Diagrams over prose — a good diagram replaces 3 pages of text

4

Decision log is non-negotiable — future you will thank present you

5

Exit criteria before entry — don't start a phase without knowing what done looks like
