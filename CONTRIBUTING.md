# Contributing to AISHA

First off — thank you for taking the time to contribute. AISHA is better because of people like you.

## Ways to contribute

### 1. Write a tutorial post
The most valuable contribution. Write about something you know well in the areas of:
- Solution architecture and design
- Lean processes and documentation
- AI integration patterns
- Startup tech strategy
- Automation approaches

See [Writing posts](#writing-posts) below.

### 2. Fix bugs or typos
Found something broken or a typo? Open an issue or submit a PR directly.

### 3. Improve the design
Design improvements, accessibility fixes, and mobile optimisations are welcome.

### 4. Extend the LSF Framework
Have additions or improvements to the Lean Solution Framework? Open a discussion first.

---

## Writing posts

1. Fork the repository
2. Create your post in `_posts/YYYY-MM-DD-your-title.md`
3. Use this front matter:

```yaml
---
layout: post
title: "Your Post Title"
date: 2026-03-01
categories: [Architecture]
tags: [relevant, tags]
read_time: 8
excerpt: "One clear sentence summarising the post."
author: "Your Name"
---
```

4. Write in plain Markdown
5. Submit a Pull Request with a clear description

### Content guidelines

- **Practical over theoretical** — show, don't just tell
- **Lean** — respect the reader's time. Get to the point.
- **Opinionated but honest** — share your view, acknowledge tradeoffs
- **No AI-generated filler** — write like a human, for humans
- **Diagrams welcome** — add images to `assets/images/posts/`

---

## Pull Request process

1. Fork the repo and create a branch: `git checkout -b feature/your-feature`
2. Make your changes
3. Test locally: `bundle exec jekyll serve`
4. Commit with a clear message: `git commit -m "Add post: solution architecture vs design"`
5. Push and open a Pull Request
6. Fill out the PR template

---

## Code of Conduct

Be kind. Be constructive. Assume good intent. That's it.
